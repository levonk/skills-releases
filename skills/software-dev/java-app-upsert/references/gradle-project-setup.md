# Gradle Project Setup

Detailed guidance for scaffolding and configuring Gradle-based Java applications
using Kotlin DSL.

## Standard build.gradle.kts

```kotlin
plugins {
    java
    application
}

group = "com.example"
version = "1.0.0-SNAPSHOT"

java {
    toolchain {
        languageVersion = JavaLanguageVersion.of(17)
    }
}

repositories {
    mavenCentral()
}

application {
    mainClass = "com.example.Main"
}

tasks.test {
    useJUnitPlatform()
}
```

## Gradle Wrapper

Always commit the Gradle wrapper so builds are reproducible without a
pre-installed Gradle:

```bash
gradle wrapper --gradle-version 8.5
```

This generates:
- `gradlew` (Unix)
- `gradlew.bat` (Windows)
- `gradle/wrapper/gradle-wrapper.properties`
- `gradle/wrapper/gradle-wrapper.jar`

Pin the Gradle version in `gradle-wrapper.properties`:

```properties
distributionUrl=https\://services.gradle.org/distributions/gradle-8.5-bin.zip
distributionType=bin
```

## Version Catalogs

Use a version catalog (`gradle/libs.versions.toml`) to centralize dependency
versions — the Gradle equivalent of a Maven BOM:

```toml
[versions]
spring-boot = "3.3.0"
junit = "5.10.2"
testcontainers = "1.19.8"

[libraries]
spring-boot-starter-web = { module = "org.springframework.boot:spring-boot-starter-web", version.ref = "spring-boot" }
junit-jupiter = { module = "org.junit.jupiter:junit-jupiter", version.ref = "junit" }
testcontainers-junit = { module = "org.testcontainers:junit-jupiter", version.ref = "testcontainers" }

[plugins]
spring-boot = { id = "org.springframework.boot", version.ref = "spring-boot" }
```

Reference in `build.gradle.kts`:

```kotlin
dependencies {
    implementation(libs.spring.boot.starter.web)
    testImplementation(libs.junit.jupiter)
    testImplementation(libs.testcontainers.junit)
}
```

## Configuration Cache

Enable the configuration cache for faster incremental builds:

```properties
# gradle.properties
org.gradle.configuration-cache=true
org.gradle.configuration-cache.problems=warn
org.gradle.parallel=true
org.gradle.caching=true
```

## Multi-Project Builds

```kotlin
// settings.gradle.kts
rootProject.name = "my-app"
include("core", "api", "worker")
```

```kotlin
// build.gradle.kts (root)
plugins {
    java
}

allprojects {
    group = "com.example"
    version = "1.0.0-SNAPSHOT"
}

subprojects {
    apply(plugin = "java")

    java {
        toolchain {
            languageVersion = JavaLanguageVersion.of(17)
        }
    }

    repositories {
        mavenCentral()
    }

    tasks.test {
        useJUnitPlatform()
    }
}
```

## Kotlin DSL Tips

- Use `val` for plugin accessors generated by the `plugins {}` block.
- Reference catalog entries as `libs.<name>` (hyphens become dots).
- Use `tasks.named<Test>("test")` for typed task configuration in
  multi-project builds.

## Test Source Sets

For integration tests as a separate source set:

```kotlin
sourceSets {
    create("integrationTest") {
        java {
            srcDir("src/integration-test/java")
        }
        resources {
            srcDir("src/integration-test/resources")
        }
    }
}

val integrationTest = task<Test>("integrationTest") {
    description = "Runs integration tests."
    group = "verification"
    testClassesDirs = sourceSets["integrationTest"].output.classesDirs
    classpath = sourceSets["integrationTest"].runtimeClasspath
    useJUnitPlatform()
    shouldRunAfter(tasks.test)
}

tasks.check {
    dependsOn(integrationTest)
}
```

## Cache Volumes

In CI and dev containers, mount the Gradle cache as a volume named
`gradle-cache`:

```yaml
volumes:
  - name: gradle-cache
    temp: {}
# Mount at ~/.gradle
```

## Health Check Pattern

Verify Gradle is available with:

```bash
gradle --version
```

Or using the wrapper:

```bash
./gradlew --version
```

This confirms the Gradle version, Kotlin version, and JVM are all compatible.
