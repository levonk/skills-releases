# Concern Catalog

Each concern file in `assets/concerns/` covers one category of ignore
patterns. Add a pattern to the concern that best matches its purpose.

## secrets.ignorefile

Credentials, API keys, certificates, and sensitive files that must never be
committed or indexed. Includes paths targeted by known malware (litellm
issue #24512).

**Key patterns**: `.env`, `.aws/`, `.ssh/*`, `*.pem`, `*.key`, `.kube/config`,
`.npmrc`, `.netrc`, `*.tfstate`, `.vault/`

**Used by**: all outputs (every tool should ignore secrets)

## build-artifacts.ignorefile

Compiler/interpreter output and build tool directories. These are
regenerated from source and should never be committed or indexed.

**Key patterns**: `target/`, `dist/`, `build/`, `.next/`, `__pycache__/`,
`coverage/`, `*.class`, `*.egg-info/`, `result`, `result-*` (Nix), `tags`,
`.ctags` (ctags output)

**Used by**: all outputs

## os-files.ignorefile

OS-generated files — system metadata, thumbnails, and filesystem artifacts.
Not project content; safe to ignore everywhere.

**Key patterns**: `.DS_Store`, `Thumbs.db`, `._*`, `.Spotlight-V100`,
`Desktop.ini`

**Used by**: `.gitignore`, `.npmignore`, `.dockerignore` (not AI ignores —
AI tools don't need to see OS files but it doesn't hurt to leave them
visible)

## editor-files.ignorefile

Per-developer IDE and editor settings that vary by machine. Not shared
project content.

**Key patterns**: `.idea/`, `*.swp`, `*.sublime-*`, `*.iml`,
`.vscode/settings.json`, `*.code-workspace`

**Note**: `.vscode/settings.json` is ignored (per-user), but shared
workspace settings in `.vscode/extensions.json` or `.code-workspace`
settings blocks are NOT ignored (they're shared).

**Used by**: `.gitignore`, `.npmignore`, `.dockerignore` (not AI ignores —
same reasoning as os-files)

## dependencies.ignorefile

Third-party packages installed by package managers. Regenerated from
lockfiles; never committed or indexed.

**Key patterns**: `node_modules/`, `.venv/`, `venv/`, `.gradle/`,
`packages/`, `.m2/`

**Used by**: all outputs

## ai-generated.ignorefile

Files generated by AI coding assistants and agents. Per-session or
per-tool artifacts, not shared project content.

**Key patterns**: `.claude`, `.cursor`, `.cursorignore`, `.windsurf`,
`.codeiumignore`, `.codegraph/`, `.archon/`, `generated-code/`

**Used by**: `.codeiumignore`, `.cursorignore`, `.aiexclude`, `.npmignore`,
`.dockerignore`, VS Code settings. NOT in `.gitignore` by default — AI
tool dirs like `.claude` are often committed so collaborators share AI
guidance. Add `ai-generated` to `.gitignore` concerns for repos where AI
exhaust should not be committed.

## dev-local.ignorefile

Per-developer overrides and local environment state. Not shared; varies by
machine.

**Key patterns**: `*.local.*` (any extension), `.devbox/`, `.direnv/`,
`.cache/`, `.local/`, `.cflare/`, `.wrangler/`, `.obsidian/`, Ansible local
overrides, Vagrant local files

**Used by**: all outputs

## binaries.ignorefile

Compiled executables, archives, media, and other non-text files. These
bloat repos and can't be diffed meaningfully.

**Key patterns**: `*.o`, `*.exe`, `*.dll`, `*.zip`, `*.png`, `*.mp3`,
`*.pdf`, `*.ttf`, `*.dmg`

**Used by**: all outputs

## vcs-meta.ignorefile

Internal state of version control systems. Never committed (except
.gitignore itself) or indexed.

**Key patterns**: `.git/`, `.hg/`, `.svn/`, `.jj/`

**Used by**: all outputs

## logs.ignorefile

Application and server log files. Runtime output, not source content.

**Key patterns**: `*.log`, `logs/`, `__logs__/`, `npm-debug.log*`

**Used by**: all outputs

## lockfiles.ignorefile

Package manager lockfiles that MUST be committed for reproducible builds but
should not litter IDE search results or trigger file watchers. This is a
special concern: it is NOT included in `.gitignore` or other gitignore-syntax
outputs (lockfiles must be checked in). It IS included in VS Code
`search.exclude`, `files.watcherExclude`, and ripgrep `ripgrepignore`.

**Key patterns**: `*.lock`, `package-lock.json`, `pnpm-lock.yaml`, `yarn.lock`,
`Cargo.lock`, `poetry.lock`, `uv.lock`, `flake.lock`, `go.sum`

**Used by**: `vscode-search-exclude`, `vscode-watcher-exclude`,
`ripgrep-ignore`. NOT in `.gitignore`, `.codeiumignore`, `.dockerignore`, etc.

## When to Add a Pattern

1. **Is it a secret?** → `secrets.ignorefile`
2. **Is it generated by a build tool?** → `build-artifacts.ignorefile`
3. **Is it created by the OS?** → `os-files.ignorefile`
4. **Is it an IDE/editor settings file?** → `editor-files.ignorefile`
5. **Is it a third-party dependency?** → `dependencies.ignorefile`
6. **Is it created by an AI tool?** → `ai-generated.ignorefile`
7. **Is it a local override or env wrapper?** → `dev-local.ignorefile`
8. **Is it a binary/media file?** → `binaries.ignorefile`
9. **Is it VCS internal state?** → `vcs-meta.ignorefile`
10. **Is it a log file?** → `logs.ignorefile`
11. **Is it a lockfile that must be committed but not searched?** → `lockfiles.ignorefile`

If a pattern doesn't fit any category, consider whether a new concern is
needed, or default to `dev-local.ignorefile` (the catch-all for
per-developer files).
