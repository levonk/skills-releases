# Snippets

## Definition

Snippets are generated code fragments for specific features. They are
templates that produce boilerplate code when a feature is activated.
Snippets are the most granular primitive — small, focused code files
rather than markdown guidance.

## Primary Role

**Code generation** — a snippet provides boilerplate code that would be
tedious to write manually, generated from a template when a feature is
activated.

## Scope

Narrowest — a single code file (TypeScript, Python, etc.).

## Frontmatter

Snippets typically have **no frontmatter**. They are code template files
(`.tmpl`) that render to actual code files.

## Body Structure

Snippets are code files, not markdown:

```typescript
// Example: src/domain/types/user-id.ts
// Generated snippet for feature: domain-primitives-branded-types@v1

import type { Brand } from './brand';
import { brand } from './brand';

export type UserId = Brand<string, 'UserId'>;

export function UserId(value: string): UserId {
  return brand<string, 'UserId'>(value);
}
```

## Loading Behavior

Generated on-demand when a feature is activated. Not loaded into context.

## Reusability

Reusable across projects — the same snippet template can generate code for
any project that activates the feature.

## Autonomy Level Changes

No — snippets are code, not autonomous.

## Personality/Behavior

No — snippets are code templates, no personality.

## Planning and Reasoning

No — snippets are static templates. No planning.

## File Location

`src/current/snippets/features/<feature-name>/<language>/<file>.tmpl`

## How Snippets Are Used

1. Feature activation (e.g., `domain-primitives-branded-types@v1`)
2. Workflow generates snippets for the target language/framework
3. Snippets are placed in the appropriate project directory
4. Developer customizes the generated code

## Snippet vs Template

| Aspect | Snippets | Templates |
|--------|----------|-----------|
| Output | Code files (TypeScript, Python) | Markdown structures |
| Purpose | Generated boilerplate | Reusable document structure |
| Trigger | Feature activation | Workflow/skill/agent reference |
| Format | Code templates (`.tmpl`) | Markdown templates (`.md.tmpl`) |

## Examples

- `snippets/features/domain-primitives-branded-types@v1/typescript/user-id.ts.tmpl`
- `snippets/features/domain-primitives-branded-types@v1/typescript/brand.ts.tmpl`

## Producer Skill

No dedicated `snippet-upsert` skill. Snippets are authored by hand or
generated by feature workflows. Use `ai-guidance-improver` to audit them.
